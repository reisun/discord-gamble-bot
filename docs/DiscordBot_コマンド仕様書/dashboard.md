# `/dashboard` - Webアプリへのリンクを表示する

## 説明
Webアプリにアクセスするためのワンタイムトークン付きリンクをボタン形式で表示する。
管理者ロールを持つ場合は編集権限付き（editor）、一般ユーザーは閲覧権限（viewer）のトークンを発行する。
URLはボタンに隠蔽され、Discord上にそのまま表示されない。

## オプション
なし

## 処理フロー

1. 実行されたサーバーの guildId を取得
2. メンバーの管理者ロールを確認し、role を決定（editor または viewer）
3. サーバー API（`POST /api/auth/token`）にてトークンを生成
4. URL構築: `${WEB_APP_BASE_URL}/#/dashboard/${guildId}?token=${rawToken}`
5. ボタンコンポーネントを Ephemeral で返信（URL は表示しない）

## ボタン動作

ボタンクリック時のアクセス先: `${WEB_APP_BASE_URL}/#/dashboard/${guildId}?token=XXXX`

- ハッシュフラグメント内にトークンを埋め込むため、サーバーのアクセスログには残らない
- Web アプリがトークンをハッシュから取得し、API呼び出しの Bearer 認証に使用する
- ユーザー情報（名前・アバター）を返す WebAPI でトークンの有効期限を検証する
- 期限切れの場合、API は `401 TOKEN_EXPIRED` を返す → Web アプリが再入手を案内する

## トークン有効期限

| ロール | 有効期限 |
|--------|---------|
| editor（管理者） | 12時間 |
| viewer（一般ユーザー） | 12時間 |

## 応答メッセージ（Ephemeral）

管理者の場合:
```
🔑 管理者用のリンクです。このボタンは本人のみ使用してください。有効期限は12時間です。
[Webアプリを開く] ← ボタン
```

一般ユーザーの場合:
```
🌐 閲覧用のリンクです。このボタンは本人のみ使用してください。有効期限は12時間です。
[Webアプリを開く] ← ボタン
```

## エラー条件

| 条件 | 応答 |
|------|------|
| DM（サーバー外）から実行 | 「サーバー内でのみ使用できます。」（Ephemeral） |
| URL 設定未完了 | 「URL設定が不足しています。管理者に確認してください。」（Ephemeral） |
| トークン生成失敗 | 「アクセストークンの生成に失敗しました。」（Ephemeral） |
