# `/link` - Webアプリへのリンクを表示する

## 説明
Webアプリにアクセスするためのワンタイムトークン付きリンクをボタン形式で表示する。
管理者ロールを持つ場合は編集権限付き（editor）、一般ユーザーは閲覧権限（viewer）のトークンを発行する。

## オプション
なし

## 処理フロー

1. 実行されたサーバーの guildId を取得
2. メンバーの管理者ロールを確認し、role を決定（editor または viewer）
3. サーバー API（POST /api/auth/token）にてワンタイムトークンを生成
4. ボタンコンポーネントを Ephemeral で返信（URL は表示しない）

## ボタン動作

ボタンクリック時のアクセス先: `${SERVER_BASE_URL}/${guildId}?token=XXXX`

サーバー側での処理:
1. トークンを SHA-256 ハッシュで検証（有効期限5分・未使用・guildId一致）
2. 検証 OK → トークンを使用済みにマーク → セッション発行 → `/[guildId]` へ302リダイレクト
3. 以後はセッション Cookie で認証

## セッション有効期限

| ロール | 有効期限 |
|--------|---------|
| editor（管理者） | 2時間 |
| viewer（一般ユーザー） | 24時間 |

## 応答メッセージ（Ephemeral）

管理者の場合:
```
🔑 管理者用のリンクです。このボタンは本人のみ使用してください。有効期限は5分です。
[Webアプリを開く] ← ボタン
```

一般ユーザーの場合:
```
🌐 閲覧用のリンクです。このボタンは本人のみ使用してください。有効期限は5分です。
[Webアプリを開く] ← ボタン
```

## エラー条件

| 条件 | 応答 |
|------|------|
| DM（サーバー外）から実行 | 「このコマンドはサーバー内でのみ使用できます。」（Ephemeral） |
| URL 設定未完了 | 「URL設定が不足しています。管理者に確認してください。」（Ephemeral） |
| トークン生成失敗 | 「アクセストークンの生成に失敗しました。」（Ephemeral） |
