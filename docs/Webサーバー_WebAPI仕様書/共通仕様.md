# Webサーバー WebAPI 仕様書 - 共通仕様

## ベースURL
```
http://localhost:<PORT>/api
```

## 認証
管理者操作が必要なエンドポイントはリクエストヘッダーまたはクエリパラメータにトークンを付与する。

```
Authorization: Bearer <token>
# または
?token=<token>
```

トークンが不正または未指定の場合は `401 Unauthorized` を返す。  
管理者権限が必要な操作に一般ユーザーがアクセスした場合は `403 Forbidden` を返す。

### 内部 API（Bot → Server）

Bot が自身の認証トークンを取得するための専用エンドポイントとして `/internal/api/auth/token` を用意している。

| パス | 用途 | 認証 | nginx 公開 |
|---|---|---|---|
| `/api/...` | Web UI・Bot 共用 | DBトークン必須 | ○（プロキシ対象） |
| `/internal/api/auth/token` | Bot のトークン取得専用 | 不要（管理者扱い） | ×（非公開） |

- nginx は `/api` のみをプロキシするため、`/internal/...` は外部からアクセス不可
- Bot は起動時に `/internal/api/auth/token` で editor トークンを取得し、以降は取得したトークンで公開 `/api/...` を使用する
- トークンは有効期限（12時間）前に自動更新される

## レスポンス形式

**成功時**
```json
{
  "data": { ... }
}
```

**エラー時**
```json
{
  "error": {
    "code": "ERROR_CODE",
    "message": "エラーメッセージ"
  }
}
```

## 共通エラーコード

| ステータス | コード | 説明 |
|-----------|--------|------|
| 400 | `VALIDATION_ERROR` | リクエストパラメータ不正 |
| 401 | `UNAUTHORIZED` | 認証トークン不正 |
| 403 | `FORBIDDEN` | 権限不足 |
| 404 | `NOT_FOUND` | リソースが存在しない |
| 409 | `CONFLICT` | 状態が操作を許可しない（例: 締め切り後の賭け） |
| 500 | `INTERNAL_ERROR` | サーバー内部エラー |
