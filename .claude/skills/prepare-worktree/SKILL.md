---
name: prepare-worktree
description: 対象タスクに使う worktree を確認し、必要なら作成して作業開始可能な状態にする
---

対象: $ARGUMENTS

目的:
対象タスクに対して、既存 worktree を使うか新規作成するか判断し、作業開始可能な状態を整える。

手順:
1. 対象タスクの目的と作業内容を確認する
2. 既存 worktree で流用可能なものがあるか確認する
3. 既存 worktree が不適切なら、新規 worktree 作成が必要と判断する
4. 必要なら適切な branch 名を決める
5. worktree を作成または選択し、対象 branch を checkout する
6. 作業ディレクトリの状態を確認する
   - 対象 branch が正しいか
   - 意図しない未コミット変更がないか
   - 必要なファイルが揃っているか
7. 必要に応じて依存関係や初期化手順の実施要否を確認する
8. この worktree台帳.md を活用し作業開始可能か判断する

判断基準:
- 調査だけなら既存 worktree を優先してよい
- 修正作業は専用 worktree を優先する
- 並列作業や別案比較を行う場合は新規 worktree を優先する
- 既存 worktree に無関係な変更や別タスクの変更が混在している場合は使わない

出力:
- 対象タスク
- 使用する worktree
- branch
- 新規作成の有無
- 作業開始可否
- 必要な注意点
