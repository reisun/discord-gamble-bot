name: deploy-after-approval
description: 承認済みPRまたは承認済み変更を、条件確認のうえでデプロイするときに使う
disable-model-invocation: true
---

対象: $ARGUMENTS

目的:
承認済み変更について、デプロイ条件を確認した上で安全にデプロイする。

手順:
1. 対象変更が承認済みであることを確認する
2. デプロイ対象の環境を確認する
   - local: なし
   - staging: なし
   - production:
      Webアプリ: GitHub Pages上
      WebAPI: ローカル環境で立ち上げた Docker Compose
      Bot   : ローカル環境で立ち上げた Docker Compose
3. 次を確認する
   - 対象ブランチが正しいか
   - 想定外の未コミット変更がないか
   - 必要なテスト、lint、ビルドが通っているか
   - 必要な設定値やシークレットが揃っているか
   - デプロイされたバージョンが正しいか
4. デプロイ手順を短く要約する
5. 高リスク環境の場合は再確認事項を先に出す
6. デプロイを実行する
7. 実行結果を確認する
8. 必要ならロールバック条件や初動確認項目を示す

ルール:
- 承認未了の変更はデプロイしない
- production デプロイは確認事項を省略しない
- デプロイ失敗時は原因候補と次の確認項目を整理する

出力:
- デプロイ対象
- 環境
- 事前確認結果
- 実行内容
- デプロイバージョン
- 結果
- 懸念点
- 必要ならロールバック案
