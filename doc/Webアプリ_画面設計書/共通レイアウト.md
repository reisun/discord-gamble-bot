# Webアプリ 共通レイアウト

```
┌─────────────────────────────────────────────────────────────┐
│  ヘッダー: [サーバーアイコン※] 賭けダッシュボード [権限バッジ]│
├─────────────────────────────────────────────────────────────┤
│  メインエリア                                               │
│  パンくずリスト                                             │
│  ─────────────────────────────────────────────────────────  │
│                                                             │
│  (各画面のコンテンツ)                                       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

※ サーバーアイコン: `guilds.guild_icon_hash` をDBに保存し、`https://cdn.discordapp.com/icons/{guildId}/{hash}.png` で表示。未設定時はデフォルトのコインロゴを表示。

## ルーティング方式
GitHub Pages でホスティングするため、**ハッシュルーティング**（Hash Routing）を採用する。  
すべての画面 URL は `#` 以降で表現され、サーバーへのリクエストは発生しない。

```
https://<github-pages-url>/#/dashboard/<guildId>
https://<github-pages-url>/#/dashboard/<guildId>/1/edit
```

## 権限
| 権限 | 条件 |
|------|------|
| 管理者 | ハッシュ内クエリパラメータ `?token=<正しいトークン>` |
| 一般ユーザー | それ以外 |

トークンはハッシュフラグメント内に含めることでサーバーのアクセスログに残らない。  
例: `https://<github-pages-url>/#/dashboard/<guildId>?token=xxxxxxxxxxxxxxxx`

管理者専用要素は権限なし時に非表示とする。
